## Context

当前 CalendarHeatmap 组件存在视觉拥挤问题：每个格子包含数字 + emoji 图标 + 背景色三层信息，在移动端小屏幕上重叠。用户反馈"太奇怪、不美观"。已尝试修修补补（调 gap、去 emoji 改圆点、改背景色）但效果不佳。需要从根本上重新设计日历的视觉层级。

## Goals / Non-Goals

**Goals:**
- 日历格子只显示数字 + 可选小圆点，无拥挤
- 有日记的日期用颜色区分（数字着色），一眼可辨
- 今天的日期用实心圆突出
- 页面整体干净、年轻化

**Non-Goals:**
- 不改变情绪趋势图表
- 不改变日记列表卡片
- 不改写 AI 对话或语音功能

## Decisions

### Decision 1: 数字着色方案（方向 B）

**选择**: 通过数字颜色区分有无日记，底部小圆点作为辅助标记。

**替代方案**:
- 方向 A（纯圆点）：太冷淡，失去温暖感 — 不采纳
- 方向 B+（圆点颜色=情绪）：信息量仍然偏大，圆点太小无法分辨颜色 — 不采纳

**理由**: 方向 B 信息层级最清晰：颜色 → 再认有无日记；实心圆 → 找今天；灰色 → 安静退后。

### Decision 2: 移除 emoji 图标

**选择**: 日历格子不再显示 🌱🌸🌻🌟 等连续性图标。连续打卡信息由 StreakBadge 组件独立展示。

**理由**: 图标在 7 列网格中过于拥挤，且与 StreakBadge 重复。

### Decision 3: 容器样式简化

**选择**: 日历卡片从 `glass-card`（毛玻璃）改为 `bg-white rounded-2xl border border-black/5`。页面背景从 `warm-gradient` 改为 `bg-[#fafaf9]`。

**理由**: 毛玻璃 + 渐变底会叠加出灰蒙蒙的视觉效果，白卡在浅灰底上更干净。

## Risks / Trade-offs

- [风险] 去掉 emoji 图标后，用户可能觉得日历"太空"或"不温暖" → 缓和措施：保留暖橙色的数字着色和情绪色系，StreakBadge 保留火焰图标
- [风险] 颜色对比度在夜间模式不可用 → 当前仅支持浅色主题，后续加暗色模式时再适配
